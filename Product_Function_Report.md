# 产品功能模块报告：Freya V2 (Co-Star 风格极简版)

基于您的反馈，我们调整了设计方向。新版本将严格遵循 **"Co-Star" 式的极简主义设计哲学**：主界面保持高度清爽、神秘与留白，所有深度信息（AI 报告、详细数据、趋势图）均通过**点击/展开**的方式按需查看。

## 1. 设计哲学 (Design Philosophy)

*   **Less is More**: 首页不展示大段文字。仅保留核心结论、关键分数和极简图形。
*   **On-Demand Disclosure (按需披露)**: 用户对感兴趣的模块点击后，才通过 Bottom Sheet（底部弹窗）或二级页面展示详尽的 AI 解读和图表。
*   **Aesthetic (美学)**: 黑白/单色调为主，高对比度排版，强调字体排印（Typography）和留白。使用神秘学符号作为视觉锚点。

## 2. 核心页面架构

### 2.1 首页 (Home) - "The Daily Glimpse"
**核心体验**：像一张每日运势卡片，一眼看尽核心，点击探索深意。

| 模块 | 默认展示状态 (Default View) | 点击交互 (On Tap) |
| :--- | :--- | :--- |
| **顶部状态栏** | **时间切换器**：极简文本切换 (Today / Tomorrow / Week / Month / Year)。 | 切换整个页面的数据维度。 |
| **A. 核心运势** | **"The Headline"**：一句直击灵魂的 AI 短语（例如："Chaos is a ladder."）。<br>**综合评分**：一个醒目的数字或抽象图形。 | **打开 "AI Insight" 底部弹窗**：<br>展示完整的 AI 深度分析报告（结合本命盘 + 行运），解释今日运势背后的逻辑。 |
| **B. 五维概览** | **极简五维图**：仅展示 5 个维度（情感/财富等）的图标 + 当前分数/能量条。<br>**不显示**复杂的曲线图。 | **进入 "Energy Dimensions" 详情页**：<br>展示该时间粒度下的五维趋势曲线图 (Trend Chart)。<br>AI 对每个维度的具体分析。 |
| **C. 上天指引** | **"The Do's & The Don'ts"**：<br>仅展示：宜/忌的 **关键词 + 图标**。<br>（例如：宜 "Meditation" 🧘‍♀️ | 忌 "Impulse" ⚡️） | **展开 "Daily Guidance"**：<br>解释为什么今日宜冥想、忌冲动（结合星象原因）。<br>展示幸运数字、颜色、方位等完整 Buff 列表。 |
| **D. 天象速览** | **"Your Sky Today"**：<br>仅列出 1-2 个**核心**天象标题。<br>（例如："Mercury Retrograde" 水逆中） | **打开 "Transits" 解读**：<br>AI 解读该天象对**你个人**的具体影响（不仅仅是通用解释，而是结合你的星盘）。 |

### 2.2 星盘页 (Chart) - "The Blueprint"
**核心体验**：通过交互式星盘探索自我，数据层层递进。

| 模块 | 默认展示状态 (Default View) | 点击交互 (On Tap) |
| :--- | :--- | :--- |
| **A. 视觉星盘** | **交互式星盘图**：屏幕中央仅渲染一张精美的星盘（本命盘或行运盘）。<br>保留行星符号，隐藏复杂的相位线干扰。 | **交互模式**：<br>双指缩放查看细节。<br>点击星体高亮显示其相位连线。 |
| **B. 行星落座** | **极简列表**：仅列出：行星符号 + 落座星座符号。<br>（例如：☉ Sun → ♌️ Leo） | **展开 "Placement Details"**：<br>显示详细度数、宫位、相位列表（Trine, Square 等）。<br>展示该配置的 AI 简短解读（可选）。 |
| **C. 全量天象** | **"Cosmic Events"**：一个折叠的列表入口，显示今日天象数量（例如 "5 Events Today"）。 | **展开列表**：<br>以时间轴形式列出所有精密的天文事件（入座、逆行、相位），不带 AI 解读，纯数据展示。 |

### 2.3 AI 咨询 (Oracle)
*   **Chat Interface**: 保持极简对话框。
*   **Tools**: 塔罗牌、答案之书等工具隐藏在 "+" 号菜单中，不占用主屏空间。

### 2.4 个人中心 (Profile)
*   **Identity Card**: 极简的个人信息卡片（名字 + 太阳/月亮/上升星座符号）。
*   **Settings**: 列表式设置项。

---

## 3. 交互逻辑总结
*   **首页** = 目录/摘要 (Table of Contents)
*   **点击** = 深入阅读 (Deep Dive)
*   **所有复杂图表（如趋势线）** = 藏在二级详情页中

这种设计能确保用户在首页不会感到压力（Information Overload），同时保留了产品的专业深度。
